# Nushell Stow Management Design

## Goal

Move Nushell configuration ownership from Home Manager-generated files to the existing GNU Stow-managed `config` package, while keeping `nushell`, `zoxide`, and `oh-my-posh` installed by Nix.

## Current State

- [nix/home/nushell.nix](/Users/kevin/projects/personal/dotfiles/nix/home/nushell.nix) enables `programs.nushell`, `programs.zoxide`, and `programs.oh-my-posh`.
- Home Manager currently generates `config.nu` and `env.nu` into the Nix store and links them into `~/Library/Application Support/nushell/`.
- Custom commands and secrets already live in [config/.config/env.nu](/Users/kevin/projects/personal/dotfiles/config/.config/env.nu), which is sourced from the generated Nushell config.
- The repository already uses the `config` stow package for other application configs under `~/.config/`.

## Constraints

- Keep `nushell`, `zoxide`, and `oh-my-posh` installed by Nix.
- Do not introduce a broad `XDG_CONFIG_HOME` change for the whole machine.
- Preserve current prompt, path setup, completions, hooks, aliases, and custom `env.nu` sourcing.
- Fit the result into the current Stow workflow driven by `make install`.

## Chosen Approach

Use the existing `config` stow package to manage Nushell's macOS-native config directory directly:

```text
config/
  Library/Application Support/nushell/
    config.nu
    env.nu
```

This keeps the migration narrow:

- Nushell continues using its default macOS lookup path.
- Stow becomes the owner of the runtime config files.
- Nix keeps owning package installation only.

## Rejected Alternatives

### Move Nushell to `~/.config/nushell`

Rejected because it requires setting `XDG_CONFIG_HOME` globally, which changes lookup behavior for other tools and expands the migration blast radius.

### Keep Home Manager as a thin wrapper that sources Stow files

Rejected because the top-level Nushell config files would still be generated by Home Manager, so configuration ownership would not actually move to Stow.

## File Ownership After Migration

### Stow-managed

- `config/Library/Application Support/nushell/config.nu`
- `config/Library/Application Support/nushell/env.nu`
- existing `config/.config/env.nu`

### Nix-managed

- package installation for `nushell`
- package installation for `zoxide`
- package installation for `oh-my-posh`

### Removed from Home Manager ownership

- `programs.nushell.configFile`
- `programs.nushell.extraConfig`
- `programs.nushell.shellAliases`
- Nushell-specific integration previously emitted by `programs.zoxide` and `programs.oh-my-posh`

## Runtime Behavior

### `config.nu`

`config.nu` will own:

- color theme and `show_banner = false`
- PATH setup
- `fnm` environment import
- `NIXPKGS_ALLOW_INSECURE`
- `nuopen`
- prompt variables and `PROMPT_COMMAND`
- completions
- hooks
- aliases
- `source ~/.config/env.nu`

### `env.nu`

`env.nu` will own:

- generation of cached `zoxide init nushell` output under `~/.cache/zoxide/init.nu`
- generation of cached `oh-my-posh init nu --print` output under `~/.cache/oh-my-posh/init.nu`
- `source` of those cached scripts

This mirrors the current Home Manager behavior closely enough to avoid changing startup behavior.

## Nix Changes

- Remove the `./nushell.nix` import from [nix/home/default.nix](/Users/kevin/projects/personal/dotfiles/nix/home/default.nix).
- Delete [nix/home/nushell.nix](/Users/kevin/projects/personal/dotfiles/nix/home/nushell.nix).
- Add `nushell`, `zoxide`, and `oh-my-posh` to [nix/home/home-pkgs.nix](/Users/kevin/projects/personal/dotfiles/nix/home/home-pkgs.nix).

This preserves Nix installation while ending Home Manager's ownership of Nushell config files.

## Verification

The migration is correct when all of the following are true:

1. `stow -nv -d <repo> -t <temp-home> config` shows links for `Library/Application Support/nushell/config.nu` and `env.nu`.
2. `nu --config <repo-config> --env-config <repo-env> -c 'version | get version'` exits successfully.
3. No `programs.nushell`, `programs.zoxide`, or `programs.oh-my-posh` references remain under `nix/home/`.
4. The checked-in config still sources [config/.config/env.nu](/Users/kevin/projects/personal/dotfiles/config/.config/env.nu).
